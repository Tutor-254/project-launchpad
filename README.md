# Arcane — Open Learning Platform

Arcane is a full-stack, marketplace-style e-learning platform (Udemy-style) implementing the SRS "Open Learning Platform" specification. Instructors publish long-form video courses; students browse, enroll, and learn with progress tracking.

Design direction: **Knowledge forest** — deep emerald `#064e3b`, warm amber accents `#d97706`, Newsreader serif for headings, Inter for body copy.

## Database schema (M1)

- `profiles` — auto-created on signup, mirrors public user info (display_name, headline, bio, avatar_url)
- `user_roles` — role assignments (`student`, `instructor`, `admin`) checked via `has_role()` security-definer function
- `categories` — six seed disciplines (Development, Design, Business, Data Science, Languages, Personal Growth)
- `courses` — instructor-owned, RLS-scoped
- `course_sections` → `lectures` — curriculum tree
- `enrollments` — student ↔ course join
- `lecture_progress` — per-user, per-lecture completion + seconds watched
- `reviews` — per-user, per-course rating + comment (with `hidden` moderation flag)
- **M2:** `questions`, `answers` — course/lecture Q&A threads; `is_instructor_answer` auto-set by trigger
- **M2:** `notifications` — in-app fan-out for new answers, questions, reviews (populated by Postgres triggers)
- **M2:** `coupons` — code-based discount schema (redemption UI ships with M3)
- **M2:** `courses.search_tsv` — generated `tsvector` (title + subtitle + description) with GIN index for catalog search

All tables have `authenticated`+`service_role` grants, RLS enabled, and policies scoped via `auth.uid()` or `has_role()`.

**Storage buckets** (private):

- `course-thumbnails` — signed reads for everyone; writes gated to the owning instructor's folder
- `course-videos` — signed reads generated by the app for enrolled learners; writes gated to the owning instructor's folder

## Routes

| --- | --- |
| `/` | Marketing home + category and featured-course browse |
| `/courses` | Full catalogue, filterable by category / level, search by title |

**Secrets required for live M-Pesa** (Daraja app at https://developer.safaricom.co.ke):
`MPESA_CONSUMER_KEY`, `MPESA_CONSUMER_SECRET`, `MPESA_SHORTCODE`, `MPESA_PASSKEY`, `MPESA_ENV` (`sandbox` or `production`).
`MPESA_CALLBACK_SECRET` is auto-generated; the callback URL to register with Safaricom is
`https://<your-published-domain>/api/public/mpesa/callback?token=<MPESA_CALLBACK_SECRET>`.

**Money model:** `price_cents` is treated as minor units (100 = KES 1). STK Push amounts are rounded up to the nearest whole KES. `payouts.platform_fee_cents` is snapshotted per-payout so a later fee change never mutates history. Currency is KES-only for now.

## Notes / decisions

- **Private storage buckets.** Public buckets are disallowed by policy; all reads use short-lived signed URLs (1 h TTL, refreshed on component mount).
- **Role model.** New users default to `student`. Becoming an `instructor` is a self-service action (`/teach`) that inserts a row into `user_roles` and is gated by RLS on `courses` INSERT.
- **Two FKs on `courses.instructor_id`.** Because `auth.users` cannot be joined via PostgREST, an explicit FK `courses_instructor_profile_fkey` → `profiles.id` was added and used in embed queries.
- **Progress model.** `lecture_progress` is a `(user_id, lecture_id)` upsert flipping a `completed` boolean; extending to `seconds_watched` for resume is trivial and already schema-supported.
